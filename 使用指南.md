# 📧 Cloudflare 临时邮箱 - 使用指南

## 🎯 这是干什么的？

**临时邮箱服务** - 用于接收验证码、注册账号，保护主邮箱隐私

**典型用途：**
- 注册不信任的网站
- 接收一次性验证码
- 避免垃圾邮件
- 测试邮件功能

---

## 🚀 部署步骤（5 分钟）

### 第 1 步：获取 Cloudflare 账号

1. 访问：https://dash.cloudflare.com/sign-up
2. 免费注册账号
3. 验证邮箱

### 第 2 步：获取 API Token

1. 访问：https://dash.cloudflare.com/profile/api-tokens
2. 点击 **"Create Token"**
3. 选择 **"Edit Cloudflare Workers"** 模板
4. 点击 **"Continue to summary"**
5. 点击 **"Create Token"**
6. **复制 Token**（只显示一次！）

### 第 3 步：获取 Account ID

1. 访问：https://dash.cloudflare.com/
2. 右侧边栏找到 **Account ID**（32 位字符串）
3. 复制保存

### 第 4 步：配置 GitHub Secrets

1. 访问：https://github.com/idealzhou1/cloudflare_temp_email/settings/secrets/actions

2. 添加 `CLOUDFLARE_ACCOUNT_ID`：
   - Name: `CLOUDFLARE_ACCOUNT_ID`
   - Value: 刚才复制的 Account ID
   - 点击 **"Add secret"**

3. 添加 `CLOUDFLARE_API_TOKEN`：
   - Name: `CLOUDFLARE_API_TOKEN`
   - Value: 刚才复制的 API Token
   - 点击 **"Add secret"**

### 第 5 步：触发自动部署

1. 访问：https://github.com/idealzhou1/cloudflare_temp_email/actions
2. 选择 **"Deploy Worker"** 工作流
3. 点击 **"Run workflow"**
4. 选择分支 `main`
5. 点击 **"Run workflow"**

### 第 6 步：等待部署完成

- 大约 2-5 分钟
- 绿色 ✅ 表示成功
- 点击部署日志查看 Worker URL

## 📧 如何使用临时邮箱？

### 方式 1：网页版（最简单）

部署成功后，访问 Worker URL：
```
https://your-worker-name.your-subdomain.workers.dev
```

**功能：**
- 创建临时邮箱地址
- 查看收到的邮件
- 查看验证码/激活链接
- 删除邮件

### 方式 2：API 调用

```bash
# 创建邮箱
curl -X POST https://your-worker-url.workers.dev/api/new_address \
  -H "Content-Type: application/json" \
  -d '{"name":"test"}'

# 获取邮件
curl https://your-worker-url.workers.dev/api/mails?address=test@your-domain.com
```

### 方式 3：Telegram 机器人

配置 Telegram Bot 后，邮件会推送到您的 TG：
```
/start - 开始使用
/new - 创建新邮箱
/mail - 查看邮件
```

## ⚙️ 配置说明

### 必须修改的配置（wrangler.toml）：

```toml
[vars]
JWT_SECRET = "random-secret-key-min-32-chars"  # 必须改！
DEFAULT_DOMAINS = ["example.com"]  # 至少一个域名
DOMAINS = ["example.com"]
```

### 可选配置：

```toml
[vars]
# 自定义标题
TITLE = "我的临时邮箱"

# 管理员密码
PASSWORDS = ["your-admin-password"]

# 启用邮箱密码
ENABLE_ADDRESS_PASSWORD = true

# 启用自动回复
ENABLE_AUTO_REPLY = true

# Telegram Bot
TG_MAX_ADDRESS = 5
```

## 🔒 安全提示

1. **JWT_SECRET 必须修改** - 使用随机字符串
2. **管理员密码必须设置** - 防止未授权访问
3. **不要用于违法用途** - 仅供学习和个人使用
4. **定期清理数据** - 避免存储敏感信息

## 📞 常见问题

### Q1: 部署失败怎么办？
**A:** 检查 GitHub Secrets 是否正确配置

### Q2: 无法创建邮箱？
**A:** 检查 `DEFAULT_DOMAINS` 配置

### Q3: 收不到邮件？
**A:** 检查域名 DNS 配置（MX 记录）

### Q4: 可以绑定自己的域名吗？
**A:** 可以！在 Cloudflare Dashboard 添加自定义域名
